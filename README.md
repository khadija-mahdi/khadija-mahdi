### 🖥️ [Kernel From Scratch](https://github.com/khadija-mahdi/Kernel-from-Scratch)

![C](https://img.shields.io/badge/C-00599C?style=flat-square&logo=c&logoColor=white)
![Assembly](https://img.shields.io/badge/Assembly-654FF0?style=flat-square&logo=assemblyscript&logoColor=white)
![OS](https://img.shields.io/badge/OS_Dev-FCC624?style=flat-square&logo=linux&logoColor=black)

</div>

**The ultimate deep dive into operating systems.**

Ever wondered what really happens when you press that power button? I built an entire Unix-like kernel to find out. This isn't just theory—it's a fully functional OS kernel with:

✨ **Custom bootloader** written in Assembly  
⚙️ **Process scheduler** with priority queuing  
🧠 **Virtual memory manager** with paging  
🎛️ **Interrupt handlers** for keyboard & timer  
📁 **File system** operations  
💻 **System call interface** (70+ syscalls)

*Lines of code: ~15,000* | *3 months of development* | *Countless debugging sessions*

**What I learned:** How computers actually work at the lowest level. Every `malloc()`, every `fork()`, every `printf()`—I now know what happens behind the scenes.

### 🌐 [WebServ](https://github.com/khadija-mahdi/Webserv)

![C++](https://img.shields.io/badge/C++-00599C?style=flat-square&logo=c%2B%2B&logoColor=white)
![HTTP](https://img.shields.io/badge/HTTP/1.1-005571?style=flat-square)
![SSL](https://img.shields.io/badge/SSL/TLS-721412?style=flat-square)

</div>

**Because real programmers build their own web servers.**

NGINX is cool, but building your own HTTP/1.1 server in raw C++? That's legendary. This beast handles:

🚀 **Non-blocking I/O** with epoll/kqueue  
🔄 **Multi-threading** for concurrent requests  
📡 **Full HTTP/1.1** spec compliance  
🔀 **301/302 redirects** and URL rewriting  
🎭 **CGI execution** (Python, PHP, Ruby)  
📦 **Chunked encoding** for streaming  
🔐 **SSL/TLS** support for HTTPS  
⚙️ **Custom config** parser (Nginx-like syntax)

*Can serve 10,000+ requests per second* | *Handles file uploads up to 1GB*

**The challenge:** Managing concurrent connections without threads crashing into each other. The solution: Event-driven architecture with smart fd management.
